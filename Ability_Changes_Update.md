# Ability Upgrade Changes — Sakiri

> **Note:** Materials differ per character. **Beetle Coins stay the same across all characters.** Costs below are per level, applied to **all 4 abilities**.

---

## Feature Updates

- **Slider for ability skills** — Let users see how many resources they need to reach the next levels. The slider tracks the current level pulled from the character information.
- **Ability skill level prompt** — On the character selection screen, ask what level the current ability skills are at. Save this answer and don't ask again, but keep it **editable** for misclicks.
- **Gate ability levels behind character level** — Users can't select current ability skill levels without selecting character level first, since skill level is capped by ascension.
- **Cap ability levels by ascension** — Once character level is chosen, cap the available ability level options based on ascension.
- **Move the ascension prompt** — Remove the ascension question from character leveling. Instead, after the user states their character level (and ability skills), ask their ascension to determine the input cap.

### First-time character flow

1. Click a character for the first time → ask: **"What level is this character?"** or **"I don't have this character."**
2. Based on level, ask **current ascension** — only asked at **level 20+**, giving **2 options**:
   - Level 20 → Ascension 0 (no ascension) or Ascension 1 (one ahead)
   - Level 60 → Ascension 4 or Ascension 5
3. Reflect the selected ascension in character info as **stars (0–6)** alongside level.
4. Reflect it in character leveling too — if they're one ahead, mark that ascension cost as **already done** and remove it from the calculation.
